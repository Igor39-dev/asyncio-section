# asyncio-section

Проект для загрузки бюллетеней СПбМТСБ, парсинга торговых данных и сохранения результатов в PostgreSQL. В репозитории есть синхронный и асинхронный пайплайны, а также бенчмарк для сравнения их производительности.

## Требования

- Docker и Docker Compose
- Файл `.env` в корне проекта (параметры подключения к БД)

## Запуск через Docker Compose

1. Склонировать репозиторий:

```sh 
git clone https://github.com/Igor39-dev/asyncio-section.git
```

2. Поднимаем только PostgreSQL(13.0-alpine):

```sh
docker compose up -d db
```

3. Проверить, что БД готова:

```sh
docker compose ps
```

3. Запускать нужный пайплайн вручную командами ниже через контейнер `app`.

## Параметры запуска

### Для асинхронного и синхронного пайплайна

- `--pages` — количество страниц со ссылками (по умолчанию: все страницы)
- `--limit` — ограничение количества файлов для обработки (по умолчанию: без ограничения)
- `--batch-size` — размер батча для upsert в БД (по умолчанию: `1000`)

### Для бенчмарка

- `--pages` — количество страниц для сбора ссылок (по умолчанию: `1`)
- `--limit` — количество файлов для сравнения (по умолчанию: `10`)
- `--batch-size` — размер батча для upsert в БД (по умолчанию: `1000`)

## Команды запуска

### Асинхронный пайплайн

```sh
docker compose run --rm app python -m src.run_async_pipeline --pages 1 --limit 10 --batch-size 1000
```

### Синхронный пайплайн

```sh
docker compose run --rm app python -m src.run_sync_pipeline --pages 1 --limit 10 --batch-size 1000
```

### Бенчмарк sync vs async

```sh
docker compose run --rm app python -m tests.benchmark_pipelines --pages 1 --limit 10 --batch-size 1000
```
